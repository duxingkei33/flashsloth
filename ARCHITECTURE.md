# FlashSloth — 个人数字资产全聚合平台

> 原名：FlashSloth
> 定位：个人数字资产全聚合平台
> 愿景：一个后台管理你在互联网上的所有数字资产 — 文章、视频、商品、账号

---

## 一、统一工作流引擎（三大模块共享）

所有内容类型都走同一套流水线，差异只在**对象模型**和**编译规则**：

```
                    ┌─────────────────────────────────────┐
                    │          统一工作流引擎               │
                    └─────────────────────────────────────┘
                              │
    ┌──────────┬──────────────┼──────────────┬──────────────┐
    ▼          ▼              ▼              ▼              ▼
 ┌──────┐ ┌──────┐ ┌──────────┐ ┌──────────┐ ┌──────────────┐
 │ 采集  │→│ 编译  │→│  预览    │→│  存草稿   │→│   发布       │
 │Collect│ │Compile│ │ Preview │ │Save Draft│ │ Publish      │
 └──────┘ └──────┘ └──────────┘ └──────────┘ └──────┬───────┘
   │         │                                       │
   │    ┌────┴──────────┐                       ┌───┴────┐
   │    │ 对象类型决定    │                       │ 多平台  │
   │    │ 编译规则       │                       │ 分发     │
   │    └───────────────┘                       └────────┘
   │
   ├── 文章采集  → 标题/正文/标签/封面
   ├── 视频采集  → 脚本/素材/字幕
   └── 商品采集  → 标题/价格/描述/图片

```

### 对象模型差异（同一接口，不同实现）

| 阶段 | 📝 文章模块 | 🎬 视频模块 | 🛒 购物模块 |
|------|------------|------------|------------|
| **数据模型** | `Article` | `Video` (新建) | `Product` (新建) |
| **采集来源** | 论坛/API/手动 | B站/YouTube/本地 | 闲鱼/淘宝/监控 |
| **编译引擎** | MD→IR→BBCode/HTML | 剧本→转码→字幕→打包 | 商品描述→图片→定价 |
| **预览方式** | HTML渲染 | 播放器预览 | 商品卡片预览 |
| **草稿存储** | 数据库+本地Markdown | 数据库+本地视频 | 数据库+商品快照 |
| **发布目标** | 论坛/博客/公众号 | B站/YouTube/小红书 | 闲鱼/淘宝 |
| **撤回** | 论坛撤回/删除 | 视频下架 | 商品下架 |
| **签到** | 论坛签到 | 平台签到(如果有) | ❌ 闲鱼无签到 |

## 三、Gateway API 网关设计

### 3.1 为什么做 API 网关

当前 FS 只有**Web页面操作**和**命令行CLI**，如果要让：
- 你的个人AI助手调用FS能力
- 外部脚本/自动化工具集成
- 手机APP远程控制

就需要一层 **RESTful API 网关**。

### 3.2 API 端点设计

```
/api/v2/
├── system/
│   ├── GET    /status      系统状态 (CPU/内存/运行时间)
│   ├── POST   /restart     重启服务
│   ├── POST   /reload      重载配置
│   └── GET    /log         查看日志
│
├── accounts/
│   ├── GET    /            列出所有账号
│   ├── POST   /test       测试账号登录
│   └── POST   /refresh    刷新全部Cookie
│
├── article/
│   ├── POST   /compile     编译文章(MD→平台格式)
│   ├── POST   /preview     预览
│   ├── POST   /publish     发布
│   ├── POST   /retract     撤回
│   └── GET    /list        文章列表
│
├── video/
│   ├── POST   /compile     编译视频
│   ├── POST   /upload      上传
│   ├── POST   /publish     发布
│   ├── POST   /retract     下架
│   └── GET    /list        视频列表
│
├── product/
│   ├── GET    /search      搜索商品
│   ├── GET    /detail      商品详情
│   ├── POST   /monitor     注册价格监控
│   ├── GET    /alerts      获取价格报警
│   └── POST   /buy         自动下单(预留)
│
├── signin/
│   ├── POST   /run_all     全部签到
│   ├── GET    /status      签到状态
│   └── POST   /account     单账号签到
│
└── agent/
    ├── POST   /chat        AI助手对话
    └── GET    /capabilities 能力清单
```

### 3.3 认证方式

```
API Key + Timestamp + Signature:
1. 后台生成 API Key / Secret
2. 请求头: X-API-Key + X-Timestamp + X-Sign
3. Signature = HMAC-SHA256(Secret, method+path+timestamp+body)
4. 服务端校验签名+时间窗口(±300秒)
```

## 五、里程碑路线图

```
□ Gateway API 网关 (routes/api_v2.py + 认证)
□ 系统状态/重启/重载/日志 (Web界面)
□ 通知系统 (core/notifier.py)
□ API Key管理页面
```

### Phase 1 — 购物模块（1-2周）
```
□ 闲鱼搜索→商品卡片→比价 (Web UI)
□ 价格监控→通知推送
□ 商品数据模型 (core/product.py)
```

### Phase 2 — 视频模块（2-3周）
```
□ 视频数据模型 (core/video.py)
□ B站发布器增强→Playwright上传
□ 视频编译流水线 (core/video_compiler.py)
□ 多平台分发
```

### Phase 3 — AI增强（持续）
```
□ renhua人话集成到编译器输出
□ AI自动写文章/帖子/回复
□ AI逛闲鱼→发现好商品→推荐
□ 个人写作风格学习
□ 语音/对话式操作
```
